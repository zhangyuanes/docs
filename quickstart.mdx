## 设置您的开发环境

了解如何在本地更新您的文档并将其部署到公共环境。

### 编辑和预览

<AccordionGroup>
  <Accordion icon="github" title="在本地克隆您的文档">
    在入门过程中，我们在您的 Github 上创建了一个包含您文档内容的仓库。您可以在我们的
    [仪表板](https://dashboard.mintlify.com) 上找到此仓库。要在本地克隆仓库，请在终端中按照这些
    [说明](https://docs.github.com/en/repositories/creating-and-managing-repositories/cloning-a-repository)
    操作。
  </Accordion>
  <Accordion icon="rectangle-terminal" title="预览更改">
    预览可以帮助您确保更改看起来符合预期。我们构建了一个命令行界面来在本地渲染这些更改。 
    1. 安装
    [Mintlify CLI](https://www.npmjs.com/package/mintlify) 以在本地预览
    文档更改，使用此命令：``` npm i -g mint ```
    2. 在文档的根目录（`docs.json` 所在的位置）运行以下命令：``` mint dev ```
    <Note>
      如果您当前使用的是旧版 ```mint.json``` 配置文件，请更新 Mintlify CLI：


      ```npm i -g mintlify@latest```
      并在您的文档仓库中运行新的升级命令：

      ```mintlify upgrade```
      您现在应该使用新的 ```docs.json``` 配置文件。可以随时从您的仓库中删除 ```mint.json``` 文件。
    </Note>
  </Accordion>
</AccordionGroup>

### 部署您的更改

<AccordionGroup>

<Accordion icon="message-bot" title="安装我们的 Github 应用">
  我们的 Github 应用会自动将您的更改部署到您的文档站点，因此您
  不需要自己管理部署。您可以在
  [仪表板](https://dashboard.mintlify.com) 上找到安装链接。一旦机器人成功
  安装，仓库的提交哈希旁边应该会有一个复选标记。
</Accordion>
<Accordion icon="rocket" title="推送您的更改">
  [将您的更改提交并推送到
  Git](https://docs.github.com/en/get-started/using-git/pushing-commits-to-a-remote-repository#about-git-push)
  以便在您的文档站点中更新更改。如果您推送后没有看到
  Github 应用成功部署您的更改，您也可以通过我们的 [仪表板](https://dashboard.mintlify.com) 手动
  更新您的文档。
</Accordion>

</AccordionGroup>
